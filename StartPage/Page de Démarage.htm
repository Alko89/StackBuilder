Frequently Asked Questions:


What kind of problems can StackBuilder help me to solve?
StackBuilder is aimed at solving homogeneous stacking problems, which means it can ONLY solve problems where a single item (box/case) is involved.
Also, for the moment, it builds solutions using homogeneous layers in which the vertical axis remains the same. It can not provide solutions in which a "turned" case is used to fill remaining space.

How can I change unit system?
Go to menu "Tools\Settings"
In the Options dialog that opens, select "Options\Unit System".
Select the system you would like to use in the "Unit System" combo.
Click button OK and close StackBuilder. Restart StackBuilder so that change is taken into account.
How can I generate a report?
Once you have an analysis, select a solution by either checking it in the "Selected" column of the solutions array or pressing the "Select" button on the analysis form.
A small light bulb icon should appear under the analysis node of the StackBuilder tree.
Right-click the light bulb icon,
In the context menu that appears, select "Generate Word Report for..."
Microsoft Word should open with your report loaded.
How can I customize the report template?
The hml format report is made by transforming stackbuilder xml output to html thanks to xsl file.
The html file is then imported as a MS Word document by using word automation.
Summary : xml * xsl -> html -> doc To customize the report template, you need to:
- You need to know/learn basics of xsl,
- Go to folder: C:\Program Files (x86)\treeDiM\PLMPack\StackBuilder\ReportTemplates
- Copy and rename file ReportTemplateHtml.xsl
- Edit and modify the copied file in accordance with xml schema ReportSchema.xsd
- In StackBuilder, use menu Tools > Settings > Report and in the 'Report template' field, enter the path to the report template you have created. New reports generated with StackBuilder should be arranged as specified in the template.
How can I get to the truck/container analysis?
You need to:
- perform a case analysis first,
- select a solution using either the Select button or the solution check box,
- right-click the solution,
- select Add truck analysis...

How can I integrate StackBuilder in a workflow or an environment?
If your application can generate xml code, you could have it generate an input xml file to be processed by our executable TreeDim.StackBuilder.XmlFileProcessor.exe.
It can generate list of solutions, image, reports and StackBuilder project files.
If you have installed StackBuilder under the default folder, two example files (Input1.xml and Input2.xml) are available in the folder C:\Program Files (x86)\treeDiM\PLMPack\StackBuilder\Samples.
To see the XmlFileProcessor in action, simply run TestXmlFileProcessor.bat .
